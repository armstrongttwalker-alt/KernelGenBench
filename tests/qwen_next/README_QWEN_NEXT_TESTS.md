# Qwen Next Operator Testing

This directory contains test scripts for testing qwen_next operators with generated Triton kernels.

## Overview

The test suite provides two main testing approaches:

1. **Single Operator Test** (`test_qwen_next_operator.py`) - Test individual operators
2. **Batch Test** (`test_qwen_next_batch.py`) - Test multiple operators in a directory

Both tests:
- Read generated kernel code from `.py` files
- Use flagbench's qwen_next test framework for verification
- Output detailed test results in JSON format

## Prerequisites

- Python 3.8+
- flagbench installed and configured
- Generated kernel files (from `scripts/generate_kernel_and_verify_qwen_next.py`)

## Single Operator Test

### Usage

```bash
python tests/test_qwen_next_operator.py <kernel_file> [options]
```

### Arguments

- `kernel_file` (required): Path to the generated kernel .py file
- `--output-dir`: Output directory for test results (default: kernel_file_dir/test_results)
- `--device-count`: Number of devices to use (default: 1)
- `--timeout`: Timeout in seconds (default: 300)

### Example

```bash
# Test a single operator
python tests/test_qwen_next_operator.py \
    output/pass_at_k/round_0/aten::add.py \
    --device-count 1 \
    --timeout 300
```

### Output

The test creates:
- `test_results_{operator_name}.json` - Test results summary
- `log_0/result.json` - Detailed test report from verifier

Example output structure:
```json
{
  "operator": "add",
  "kernel_file": "/path/to/aten::add.py",
  "success": true,
  "detailed_results": {
    "op_name": "add",
    "success": true,
    "passed_tests": 10,
    "failed_tests": 0
  }
}
```

## Batch Test

### Usage

```bash
python tests/test_qwen_next_batch.py <kernel_dir> [options]
```

### Arguments

- `kernel_dir` (required): Directory containing generated kernel .py files
- `--output-dir`: Output directory for test results (default: kernel_dir/batch_test_results)
- `--pattern`: File pattern to match (default: *.py)
- `--device-count`: Number of devices per test (default: 1)
- `--timeout`: Timeout in seconds per test (default: 300)
- `--max-workers`: Number of parallel workers (default: 1)

### Examples

```bash
# Test all operators in a directory sequentially
python tests/test_qwen_next_batch.py \
    output/pass_at_k/round_0 \
    --device-count 1 \
    --timeout 300

# Test with parallel execution (4 workers)
python tests/test_qwen_next_batch.py \
    output/pass_at_k/round_0 \
    --device-count 1 \
    --max-workers 4

# Test only specific pattern
python tests/test_qwen_next_batch.py \
    output/pass_at_k/round_0 \
    --pattern "aten::*.py"
```

### Output

The batch test creates:
- `batch_test_summary.json` - Overall summary with statistics
- Individual test results in subdirectories for each operator

Example summary structure:
```json
{
  "total_operators": 94,
  "passed": 85,
  "failed": 9,
  "pass_rate": 0.904,
  "results": [...]
}
```

## Integration with Main Generation Script

These tests are designed to work with kernels generated by:
```bash
python scripts/generate_kernel_and_verify_qwen_next.py \
    --dataset qwen_next \
    --test-type triton \
    --max-rounds 10
```

The generated kernels will be in `output/pass_at_k/round_*/` directories.

## Test Framework Details

The tests use:
- **Verifier**: `sandbox.verifier.Verifier` for running tests
- **Test Module**: `src/flagbench/accuracy/test_qwen_next_ops.py` for qwen_next operator tests
- **Environment**: Automatically sets required environment variables for qwen_next testing

## Troubleshooting

### Common Issues

1. **Module not found errors**
   - Ensure you're running from the project root directory
   - Check that `src/` is in your Python path

2. **Test timeout**
   - Increase `--timeout` value for complex operators
   - Some operators may require more time for verification

3. **Device errors**
   - Ensure CUDA devices are available
   - Adjust `--device-count` based on available GPUs

## Notes

- Tests automatically configure the qwen_next test environment
- Results are saved in JSON format for easy parsing
- Failed tests include error messages for debugging
- The batch test can run in parallel for faster execution
